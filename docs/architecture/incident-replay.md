# Incident Replay And Reliability Gate Architecture

This lab turns the repository from static configuration into a repeatable
debugging and reliability-gate exercise. It replays AI inference traces for
normal traffic, cache-miss latency, dependency timeout, rollout regression, and
collector queue pressure scenarios, then evaluates whether the generated
signals match SRE expectations.

```mermaid
flowchart LR
replay["demo/incident_replay.py"] --> otlp["OTLP/HTTP traces"]
replay --> payloads["per-scenario OTLP payload JSON"]
otlp --> collector["OpenTelemetry Collector"]
collector --> debug["Debug output"]
collector --> jaeger["Optional Jaeger UI"]
replay --> report["out/incident-replay/report.md"]
report --> gate["demo/reliability_gate.py"]
gate --> gateEvidence["docs/evidence reliability gate"]
report --> capacity["demo/capacity_planner.py"]
report --> runbooks["demo/runbook_generator.py"]
payloads --> advanced["demo/advanced_reliability.py"]
report --> advanced
payloads --> detailed["demo/detailed_reliability.py"]
report --> detailed
capacity --> readiness["demo/release_readiness.py"]
runbooks --> readiness
gateEvidence --> readiness
advanced --> readiness
detailed --> readiness
report --> evidence["docs/evidence sample report and dashboard"]
```

## Signals

Each replayed trace includes:

- `incident.scenario`
- `service.version`
- `ai.inference.latency_ms`
- `cache.result`
- `sre.signal`
- dependency child spans for cache, feature store, and model inference
- `telemetry.loss_rate`
- `collector.queue.pressure`

## Why This Matters

The key portfolio claim is not just "I configured OpenTelemetry." The stronger
claim is:

> I built a runnable lab that shows how an SRE or platform engineer can isolate
> AI inference incidents using trace context, Kubernetes metadata, collector
> delivery, incident narratives, configurable SLO gates, capacity sanity checks,
> generated incident runbooks, burn-rate paging, canary rollback decisions,
> trace quality audits, alert deduplication, critical-path attribution,
> sampling evidence coverage, HPA lag analysis, tenant blast-radius checks, and
> token/GPU guardrails.

## Optional UI Path

If Docker is available:

```bash
docker compose up -d
python3 demo/incident_replay.py
open http://localhost:16686
```

Search for the `toy-ai-inference-api` service in Jaeger, then compare the
baseline and rollout regression traces.
